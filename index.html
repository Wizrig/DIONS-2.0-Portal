Execution Zones (EVM + SVM)
Dual virtual machine architecture. Deploy Solidity or Rust/C contracts. Add new zones without a hard fork.

EVM Zone

  • Engine: evmone (EVMC interface)
  • Storage: LevelDB
  • TPS: 1,000+
  • EIP-1153: Transient storage
  • EIP-2200: SSTORE tracking (9 statuses)
  • EIP-2929: Warm/cold gas metering
  • JSON-RPC: eth_blockNumber, eth_getBalance, eth_call, eth_sendTransaction

SVM Zone

  • Engine: Solana BPF interpreter
  • Account model: Lamports, executable, owner
  • PDAs: Program Derived Addresses
  • CPI: Cross-Program Invocation
  • Built-ins: System, Token, AssociatedToken, Rent
  • Crypto: SHA-256, Keccak-256, Ed25519
  • Dispute: 200-block challenge, 100 IOC bond, 10% slash
Both zones post state roots to L1 via CBlockHeader::zoneRoot. New zones can be added without a hard fork.
